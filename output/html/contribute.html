Try and Tell

You can help BOLTS by just trying it out and giving us feedback. Whether you are using FreeCAD or OpenSCAD, just install and use it and tell us

  • what didn't work
  • what was confusing for you
  • what you think could be done in a better way
  • where the documentation should be improved
  • how BOLTS could be more useful for you

There are many ways to reach us:

  • You can send mails to BOLTS@ist-dein-freund.de
  • Problems with the documentation can be reported via the comments section embedded on each page.

  • For FreeCAD related things, there is a development thread on the FreeCAD forums.
  • The BOLTS issue tracker on GitHub is also a good place to report problems and discuss improvements.

Add more parts and drawings

You can also help with adding more parts to BOLTS or improving those that are already there. To do so, you should first read about a few basic things, that will be useful to know

  • A short introduction to BOLTS
  • BOLTS development with git

Once you have done that you are prepared to contribute by

  • adding collections and classes
  • adding base functions (FreeCAD)
  • adding base modules (OpenSCAD)
  • create drawings

BOLTS scans its database for inconsistencies, common problems, missing or unclear information and automatically generates a list of things that could be fixed or improved. This is a great starting point for contributing to BOLTS
